## üéØ Brand Identity

### Brand Positioning

**Target Market:**
- Indonesian Small-Medium Businesses (10-500 employees)
- Industries: Tech startups, retail, manufacturing, services
- Tech-savvy HR managers and business owners
- Budget-conscious but quality-focused

**Value Proposition:**
> "Enterprise-grade HR management at SMB pricing, with AI that works in Bahasa Indonesia."

**Brand Pillars:**
1. **Intelligence** - AI-powered automation
2. **Simplicity** - Easy to use, quick to deploy
3. **Compliance** - Indonesian labor law compliant
4. **Affordability** - Transparent, fair pricing
5. **Local** - Built for Indonesia, by Indonesians

### Brand Personality

- **Professional yet Approachable** - Not corporate stuffy
- **Modern & Innovative** - Cutting-edge technology
- **Trustworthy** - Handling sensitive employee data
- **Empowering** - Enabling business growth
- **Local-First** - Understanding Indonesian culture

### Brand Voice & Tone

**Do's:**
- Use clear, jargon-free language
- Mix Indonesian and English naturally
- Be helpful and solution-oriented
- Show empathy for HR challenges
- Celebrate local success stories

**Don'ts:**
- Avoid overly technical language
- Don't be condescending
- Avoid excessive formality
- Don't over-promise features

## üìä Data & CMS Management Impact

### Content Management System (CMS)

**Recommended Stack:**

**Option 1: Headless CMS (Recommended)**
- **CMS:** Sanity.io or Contentful
- **Frontend:** Next.js (already using)
- **Benefits:**
  - Structured content
  - Multi-language support (ID/EN)
  - Preview capabilities
  - API-first architecture

**Option 2: Traditional CMS**
- **CMS:** WordPress with headless setup
- **Benefits:**
  - Familiar for content team
  - Rich plugin ecosystem
  - SEO plugins

### Data Structure for Website

**Content Types:**

1. **Blog Posts**
```typescript
interface BlogPost {
  id: string;
  slug: string;

  featuredImage: Image;
```

### Database Impact

**New Tables Required:**

```sql
-- Marketing content
CREATE TABLE cms_blog_posts (
  id UUID PRIMARY KEY,
  slug TEXT UNIQUE NOT NULL,
  title TEXT NOT NULL,
  content JSONB NOT NULL,
  author_id UUID REFERENCES users(id),
  category TEXT,
  tags TEXT[],
  published_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE cms_case_studies (
  id UUID PRIMARY KEY,
  slug TEXT UNIQUE NOT NULL,
  company_name TEXT NOT NULL,
  industry TEXT,
  content JSONB NOT NULL,
  metrics JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE cms_landing_pages (
  id UUID PRIMARY KEY,
  slug TEXT UNIQUE NOT NULL,
  content JSONB NOT NULL,
  status TEXT DEFAULT 'draft',
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Marketing analytics
CREATE TABLE web_analytics (
  id UUID PRIMARY KEY,
  page_path TEXT NOT NULL,
  event_type TEXT NOT NULL,
  user_id TEXT,
  session_id TEXT,
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Lead capture
CREATE TABLE leads (
  id UUID PRIMARY KEY,
  email TEXT NOT NULL,
  company_name TEXT,
  phone TEXT,
  employee_count INTEGER,
  source TEXT, -- blog, pricing, demo
  status TEXT DEFAULT 'new', -- new, contacted, converted, lost
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Product tours / demos
CREATE TABLE demo_requests (
  id UUID PRIMARY KEY,
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  company_name TEXT,
  phone TEXT,
  preferred_date TIMESTAMPTZ,
  message TEXT,
  status TEXT DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Indexes:**
```sql
CREATE INDEX idx_blog_posts_slug ON cms_blog_posts(slug);
CREATE INDEX idx_blog_posts_published ON cms_blog_posts(published_at DESC);
CREATE INDEX idx_leads_email ON leads(email);
CREATE INDEX idx_leads_status ON leads(status);
```

### Storage Impact

**Media Storage Requirements:**

**Images:**
- Blog images: ~200KB per image
- Case study images: ~500KB per study
- Feature screenshots: ~1MB per screenshot
- Logos: ~50KB per logo

**Estimated Storage (Year 1):**
- Blog posts: 100 posts √ó 5 images √ó 200KB = 100MB
- Case studies: 20 studies √ó 10 images √ó 500KB = 100MB
- Screenshots: 50 features √ó 5 screenshots √ó 1MB = 250MB
- Marketing assets: 500MB
- **Total:** ~1GB

**Video Storage:**
- Demo videos: ~100MB per video
- Tutorial videos: ~50MB per video
- **Estimated:** 2-3GB (Year 1)

**Recommendation:** Use Supabase Storage with CDN for all media assets.
